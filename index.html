Online Auction: BOT Detection

I was tasked to chase down robots for an online auction site. Human bidders on the site are becoming increasingly frustrated with their inability to win auctions vs. their software-controlled counterparts. As a result, usage from the site's core customer base is plummeting.

In order to rebuild customer happiness, the site owners need to eliminate computer-generated bidding from their auctions. The goal of this project is to identify online auction bids that are placed by 'robots', helping the site owners easily flag these users for removal from their site to prevent unfair auction activity.


APPROACH BREAKDOWN

My approach to identify these bids included:

  • 1. EDA and Data Imputation: check missing values and fill them with medians.
  • 2. Data Manipulation: change data type, feature engineering, generate new features, turn categorical data into dummy/integer variable.
  • 3. Apply machine learning models to predict bots.

The machine learning models I used were: Logistic Regression, Random Forest, Gbtree, Xgboost, Lightgbm.

The most useful features I identified were:

  • 1. bid frequency over short periods of time.
  • 2. average number of bids a user made per auction.
  • 3. the entropy for how many bids a user placed on each day of the week.
  • 4. maximum number of bids in a 20 min span.
  • 5. total number of bids placed by the user.
  • 6. minimum and median times between a user's bid and the previous bid by another user in the same auction.
  • 7. median delta-time from the last bid until the end of the auction.
  • 8. IPs used by a bidder which were also used by another user which was a bot.
  • 9. total used IPs, the most used IP, the least used IP, etc.

Spelling 'n Grammar Checker Tool

Nowadays, correction of grammatical errors is a common topic, and there are many tools available for checking spelling/grammatical errors in English, Spanish or Russian. In term of Vietnamese, there is not any tool and research for handling this topic.

Vietnamese is not easy to learn and very complicated, even both Vietnamese locals and Vietnamese learners usually make spelling/grammatical errors in the text. There are several types of error, such as spelling mistakes, using wrong words.

This project is my attempt at building a tool using machine learning to detect and fix spelling/grammatical errors in Vietnamese. This is the most challenging project I have ever done.


APPROACH BREAKDOWN

Approach to build dataset:

  • 1. Dataset of error-free sentences: web scraping from multiple open websites, news, blogs in Vietnamese.
  • 2. Dataset of error sentences: producing noises (remove, insert, replace, swap letters/words) on error-free sentences to create a dataset of errors.

First Approach:

  • 1. Idea: treat the problem as a regression prediction problem. The input is a sentence that is error-free or errors, while the output is the error rate.
  • 2. Data Manipulation: calculate the error rates, perform data cleaning for texts, tokenization.
  • 3. Train machine learning model on a pretrained PhoBert.
  • 4. Predict the error rate of sentences.

Second Approach:

  • 1. Idea: treat the problem as a text translation problem. The input is one sentence with errors, while the output is the correct/error-free sentence.
  • 2. Data Manipulation: perform data cleaning for texts, tokenization.
  • 3. Train machine learning model on a pretrained T5.
  • 4. Translate an error sentence to its correct form.

Second Approach:

  • 1. Library: Librosa, Keras.
  • 2. Extract Mel-scaled spectrogram images from audio.
  • 3. Prepare dataset with image data generator.

'Cadabra.com' - AWS App

AWS system of an E-Commerce application Cadabra.com


SYSTEM BREAKDOWN

1. Order History:

  • - Serverlogs > KDS > Lambda > DynamoDB > Client App
  • - Order history function simulate order data being generated from logs on EC2 instance and publish that data using Kinesis Data Streams into an AWS Lambda function which in turn will populate an order database in DynamoDB that the app can read from.

2. Product Recommendations:

  • - Serverlogs > KDF > S3 > EMR
  • - Automated product recommendations which involves training an ALS machine learning model based on aggregate customer purchasing data. To build this we will publish our order data through Kinesis Data Firehose into a busket S3 and spin up an AWS EMR cluster to produce recommendations model using Apache Spark.

3. Transaction Rate Alarm:

  • - Serverlogs > KDS > KDA > KDS > Lambda > SNS
  • - An real-time operational system that alerts if an unexpected rate of orders comes in all of a sudden.
